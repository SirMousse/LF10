Selektoren in CSS

Selektoren sind "Äuswähler". Sie bestimmen, welche HTML-Elemente du mit CSS gestalten möchtest.
Hier ist eine Übersicht von den Selektoren, die wir kennen müssen!

1. Übersicht: CSS Selektoren

Selektor Beispiel Beschreibung
Universalselektor * Wählt alle Elemente aus
Elementselektor p Wählt alle <p>-Tags aus
Klassenselektor .class Wählt alle Elemente aus mit der Klasse .class
ID-Selektor #id Wählt alle Elemente mit der id="id" aus
Gruppenselektor label, input Wählt alle label und alle input aus
Nachfahreselektor article p Wählt alle <p> Elemente, die sich irgendwo innerhalb eines <article> befinden.
Kindselektor section > p Wählt nur <p>, die direkt im <section> liegen
Direkte Nachbarselektor h1 + p Wählt das ERSTE <p> direkt nach einem <h1>
Allgemeine Geschwisterselektor h1 ~ p Wählt alle <p>Elemente, die nach einem <h1> auf derselben Ebene kommen.
Attributselektor input[type="password"] Hier wird nur das <input> mit dem type="password" angesprochen.

Erklärung der CSS Selektoren

1. Universalselektor: *

Was macht er? Er wählt wirklich alles aus, jedes einzelne Element auf der Seite.

Wann ist er nützlich? Vor allem am Anfang, wenn man Grundeinstellungen für die ganze Seite setzen möchte. 

                        
                            *{
                                font-family:Arial, Helvetica, sans-serif;
                                line-height: 1.5;
                                box-sizing: border-box; 
                                margin: 0;
                                padding: 0;
                            }

2. Elementselektor: body

Was macht er? Er spricht alle HTML Elemente eines bestimmten Typs an.

Wann benutzen wir ihn? Wenn alle gleichen Elemente gleich aussehen sollen. 

                        
                            body{
                                background: linear-gradient(to bottom, #eef4ff, #f8fbff);
                                color: #1f2937;
                            }

3. Klassenselektor: .class

Was macht er? Er wählt alle Elemnte aus, die dieselbe Klasse haben.

Warum ist das praktisch? Weil man damit mehrerer Elemente gemeinsam gestalten kann, auch wenn sie unterschiedliche Tags sind. 

                        
                            .wichtig{
                                background-color: yellow;
                            }

4. ID-Selektor #id

Was macht er? Er wählt genau ein Element aus mit einer bestimmten ID.

WICHTIG: Eine ID sollte nur einmal auf einer Seite vorkommen. 

                        
                            #header{
                                font-size: 2rem;        /* font-size = Schriftgröße */
                            }

5. Gruppenselektor: label, input

Was macht er? Er fasst mehrere Selektoren zusammen.

Warum ist das hilfreich? Weil du denselben CSS-Code nicht doppelt schreiben musst. 

                        
                            label, input{
                                font-size: 1rem;
                            }

6. Nachfahrenselektor: article p

Was macht er? Wählt alle pElemente, die sich irgendwo innerhalb eines article befinden.

WICHTIG: Es ist egal, ob das p direkt drin ist oder noch tiefer verschachtelt. 

                        
                            article p{
                                color: green;
                            }

7. Kindselektor: section > p

Was macht er? Er wählt nur die p Elemente aus, die direkt in einer section liegen.

WICHTIG: Nur direkte Kinder werden ausgewählt, keine Enkel oder Urenkel. 

                        
                            section > p{
                                font-weight: bold;          /* font-weight = Schriftstärke */
                            }

8. Direkte Nachbarselektor: h1 + p

Was macht er? Er wählt das ERSTE p Element direkt nach einem h1 aus.

WICHTIG: Wirklich nur das direkt folgende Element. 

                        
                            h1 + p{
                                color: red;
                            }

9. Allegmeine Geschwisterselektor: h1 ~ p

Was macht er? Er wählt alle p Elemente aus, die nach einem h1 kommen und auf derselben Ebene liegen.

Interschied zu +: Hier wird nicht nur der direkte Nachbar gewählt, sondern mehrere passende Geschwister. 

                        
                            h1 ~ p{
                                color: orange;
                            }

10. Attributselektor: input[type="password"]

Was macht er? Er wählt nur Elemente aus, die ein bestimmtes Attribut mit einem bestimmten Wert besitzen.

Wofür ist das gut? Besonders praktisch bei Formularen. 

                        
                            input[type="password"]{
                                border: 2px solid tomato;
                            }
